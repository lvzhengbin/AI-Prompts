# 🎬 Seedance Storyboard — 长视频分镜提示词生成器

将任何剧本（视频时长大于 15 秒）拆解为多个分镜提示词，在每一段视频之间保持角色形象、场景和叙事的高度一致性。

## 参考资源加载说明

本技能附带以下参考资料，在对应步骤中按需加载：

| 参考文件 | 加载时机 | 用途 |
|---------|---------|------|
| `references/video-types.md` | **第一步确认视频类型后** | 根据视频类型（动漫/科幻/广告/品牌宣传等）选择对应的提示词模板结构和节奏模式 |
| `references/visual-styles.md` | **第二步讨论视觉风格时** | 提供具体的风格关键词和搭配建议 |
| `references/color-mood.md` | **第二步讨论色调氛围时** | 提供色调 + 光影的情绪映射关系 |
| `references/shot-types.md` | **第三步构建分镜时** | 提供景别、运镜方式、转场方式的专业词汇和描述 |
| `templates/storyboard-template.md` | **第四步生成提示词时** | 包含各类场景的完整分镜提示词示例，供参考和复用 |

## 平台核心能力与限制

### 技术参数

| 输入类型 | 格式 | 数量限制 | 大小限制 |
|---------|------|---------|---------|
| **图片** | jpeg、png、webp、bmp、tiff、gif | ≤ 9 张 | < 30 MB |
| **视频** | mp4、mov | ≤ 3 个，总时长 2–15s | < 50 MB |
| **音频** | mp3、wav | ≤ 3 个，总时长 ≤ 15s | < 15 MB |
| **文本** | 自然语言 | 最大 800 汉字 | — |

**混合输入总上限：12 个文件**

### 两种生成入口

- **首尾帧入口**：上传首帧图（必须）+ 尾帧图（可选）+ 提示词文本
- **全能参考入口**：多模态组合输入（图 + 视频 + 音频 + 文本）

### ⚠️ 平台限制

- 不支持上传含有**写实真人脸部**的素材（图片和视频均不可），系统会自动拦截
- 有参考视频时生成消耗更多额度
- 视频延长时，选择的生成时长应为「新增部分」的时长
- 文本输入有最大 800 汉字限制，需精炼提示词

## 分步引导流程

### 第一步：理解用户想法

用友好的方式向用户询问以下信息（不需要逐条穷举，尽量在一次对话中完成）：

1. **你想讲什么故事？** （一句话概括核心内容）
2. **视频总时长？** （15秒以内生成单视频；超过15秒生成多段视频）
3. **横屏还是竖屏？** （默认横屏 16:9）
4. **生成模式？** （首尾帧 / 全能参考）
5. **有无参考素材？** （图片 / 视频 / 音频）

> 收到回答后，根据视频类型加载 `references/video-types.md`，了解对应类型的提示词模板结构。

---

### 第二步：深入挖掘细节

根据用户的初步回答，按需引导完善以下维度（根据情况灵活选择，不必全部提问）：

#### 1. 内容叙事
- 故事的起承转合是什么？
- 关键情节有哪些？需要哪些角色？
- 对白 / 旁白内容？

#### 2. 视觉风格
- 整体风格（参考 `references/visual-styles.md` 提供选项供用户选择）
- 色调氛围（参考 `references/color-mood.md` 提供情绪映射建议）
- 画面比例：竖屏(9:16) / 横屏(16:9) / 电影宽屏(2.35:1)

#### 3. 镜头语言
- 景别变化（参考 `references/shot-types.md` 中的景别分类）
- 运镜方式：推 / 拉 / 摇 / 移 / 跟 / 环绕 / 升降 / 希区柯克变焦？
- 转场方式：硬切 / 渐变 / 匹配剪辑 / 特效转场？

#### 4. 动作与节奏
- 主体动作：行走 / 奔跑 / 打斗 / 舞蹈 / 表情变化？
- 节奏：舒缓 / 急促 / 有张有弛？是否需要音乐卡点？

#### 5. 声音设计
- 配乐风格：激昂 / 温馨 / 悬疑 / 欢快 / 史诗？
- 音效需求：环境音 / 动作音效 / 特殊音效？
- 对白语气？

---

### 第三步：构建角色与场景基础资产

在生成分镜提示词之前，先确定「一致性锚点」——为主要角色和核心场景生成独立描述，供后续所有分镜引用。

#### 3.1 角色提示词（每个主要角色单独一条）

```
[角色名称]：[详细外观描述、服装、发型、艺术风格、光影品质标签]
```

**示例**：
```
【角色@男主】：30岁男性，西装略显疲惫，黑色短发，面容坚毅
电影写实风格，柔和侧光，高清细节
```

#### 3.2 场景/环境提示词（每个核心场景单独一条）

```
[场景名称]：[环境构成、氛围、光线、色调、艺术风格、品质标签]
```

**示例**：
```
【场景@走廊】：现代公寓长廊，夜间暖黄路灯，地面反光，安静氛围
```

> 这些角色和场景描述将作为 `@引用` 锚点，在后续所有分镜中保持视觉一致性。

---

### 第四步：构建多宫格分镜结构 和 生成分镜提示词

参考 `templates/storyboard-template.md` 中的示例格式生成提示词。

#### 4.1 时间轴规划

**单视频（≤15秒）** 的结构示例：
```
0–3秒：开场镜头，建立场景
3–6秒：发展，引入主体 / 冲突
6–10秒：高潮，核心动作 / 情感爆发
10–13秒：转折 / 过渡
13–15秒：结尾 / 落版
```

**长视频（>15秒）** 需拆解为多段，每段最长 15 秒，规则如下：
- 每段分镜独立可生成一个视频片段
- 每段之间通过**尾帧→下段首帧**实现无缝衔接

> 🔑 **核心规则**：每个分镜的最后一帧（尾帧）必须设计为该片段在剧情上的「结束定格」，使下一段可以直接 @上一段的尾帧 作为首帧，确保视频连续性。

#### 4.2 多宫格分镜总览

先生成一个「多宫格分镜总览」，展示所有片段的叙事结构：

```
【多宫格分镜总览】
@角色名 @场景名（引用第三步确定的角色和场景描述）

[分镜1 缩略描述] | [分镜2 缩略描述] | [分镜3 缩略描述]
   0–15秒           15–30秒           30–45秒

每格：关键画面 + 核心动作 + 情绪基调 + 下段衔接说明
```

#### 4.3 各分镜独立提示词

为每一个分镜生成独立的完整提示词（可直接粘贴到 Seedance 使用）：

```
【分镜 N 描述：XX到XX秒】
整体描述：@角色 @场景 风格 + 时长 + 画面比例 + 整体氛围
首帧@[上一段尾帧图片 或 角色参考图片]，尾帧@[本段结束定格图片]

0–Xs：[运镜方式]，[画面内容]，[主体动作]，[对白/旁白]，[光影/特效]
X–Ys：[运镜方式]，[画面内容]，[主体动作]，[对白/旁白]，[光影/特效]
...

【声音】配乐风格 / 音效 / 对白
【衔接说明】尾帧为 [描述]，下段首帧应 @本段尾帧 生成
```

---

### 第五步：提示词优化与使用建议

生成提示词后主动完成以下工作：

1. **平台规则检查**：确认提示词中没有违规内容（如写实真人脸部素材引用）
2. **字数检查**：单段提示词不得超过 800 汉字，如超出则精炼
3. **素材建议**：告知用户哪些素材最关键（首帧图、角色参考图等）
4. **操作指引**：说明如何在即梦平台使用 @引用语法、如何选择生成时长
5. **修改建议**：主动提出 1–2 个可优化点，邀请用户反馈

## 高级技巧

### 特殊场景提示词写法

| 场景 | 提示词写法示例 |
|------|--------------|
| **首帧控制** | `首帧@图片1，参考@图片2的角色形象` |
| **视频延长** | `将@视频1延长5s`（生成时长选「新增部分」的时长） |
| **多视频融合** | `在@视频1和@视频2之间补充场景，内容为 xxx` |
| **希区柯克变焦** | `完全参考@视频1的希区柯克变焦效果和主角面部表情` |
| **一镜到底** | `全程不切镜头，一镜到底，保持紧张节奏` |
| **角色替换** | `将@视频1中的女生换成@图片1的戏曲花旦形象` |
| **连续动作** | `角色从跳跃直接过渡到翻滚，保持动作连贯流畅` |
| **特效复刻** | `完全参考@视频1的变装特效和动作节奏` |

### 多段视频衔接策略

```
分镜1：首帧@角色图  → 生成 → 导出尾帧保存为「片段1尾帧」
分镜2：首帧@片段1尾帧，尾帧@片段2预设构图 → 生成 → 导出尾帧
分镜3：首帧@片段2尾帧 → 生成
...
```

**角色一致性三件套**：
```
@图片1 作为角色形象参考
@图片2 作为场景风格参考
@图片3 作为首帧画面
```

## 注意事项

- ⚠️ 不支持上传包含写实真人脸部的素材（图片和视频均不行）
- ⚠️ 视频参考会消耗更多生成额度
- ⚠️ 混合输入总上限是 12 个文件
- ⚠️ 单段提示词文本不超过 800 汉字
- 💡 优先上传对画面或节奏影响最大的素材
- 💡 角色一致性：建议提供多张角色参考图
- 💡 复杂运镜：用「完全参考@视频1的所有运镜效果」确保精准复刻
- 💡 视频延长时，生成长度应选「新增部分」的时长，而非总时长
